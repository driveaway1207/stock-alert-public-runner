# 五号员工报告规范

五号员工是涨停样本研究员，不是买入推荐系统。五号员工 Telegram 正文必须是“结构归因研究报告”，不是后台运行诊断报告。

## 1. 报告定位

五号员工报告只服务一个目标：从当日涨停/大涨样本中总结可复用的大级别结构、启动前端倪、共振维度和未来一号员工可优化方向。

报告正文必须写给用户看，重点回答：

- 今天3只深度样本共同体现了什么结构规律。
- 每只样本属于什么大级别结构原型。
- 启动前有哪些可量化端倪。
- 哪些维度在多只样本中共振。
- 哪些共性值得转化为一号员工后续优化因子。

## 2. Telegram 正文必须包含

- 日期与核心结论。
- 今日3只深度样本名称和选择理由。
- 每只样本的主结构原型、辅助结构原型、大级别周期。
- 核心结构线：凹口、左峰、平台上沿、头肩颈线、历史最大量K高点/实底、VBP筹码带等。
- 启动前端倪：量能恢复、平台压缩、BOLL缩口、巨震大阳、凹口攻击、攻击后盘整、右侧抬升。
- 触发确认：突破哪条核心线、实体质量、收盘位置、量能质量。
- 反证与不成立条件：不是强行套形态，证据不足必须说明。
- 3只样本共同命中的高频维度排行。
- 可转化给一号员工的优化方向。

## 3. 禁止出现在 Telegram 正文的后台字段

以下内容属于后台诊断、数据质量、运行工程信息，不得放进 Telegram 正文：

- 涨停制度分布。
- 板块制度数量分布。
- 历史K线优先级。
- 历史K线来源统计。
- K线拉取尝试多少只、失败多少只。
- source/source_count。
- 接口失败样本。
- baostock/akshare/eastmoney 运行细节。
- workflow、artifact、progress 的工程说明。

这些内容可以保留在 JSON、artifact、Actions 日志或后台诊断文件里，但不进入用户报告正文。

## 4. 维度规则

五号员工30+/60+维度必须来自可统计、可复用、可转化为模型因子的结构与资金规律。

不能计入维度的内容：

- 数据源口径。
- 运行口径。
- 纯解释口径。
- 空泛判断。
- 与未来优化、统计、总结共性无关的口径。

## 6. 后台输出位置

后台诊断字段只能放在：

- `employee5_reports/*.json`
- `employee5_progress.json`
- GitHub Actions 日志
- artifact
